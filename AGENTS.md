# AGENTS.md — Vibe Coding G2

Course monorepo with 3 independent projects + reference material (do not modify `clase-*/`).

## Structure

```
/
├── task-manager-backend/    # Express + Prisma 7 + PostgreSQL  (:3000)
├── task-manager-frontend/   # React 19 + Vite + Tailwind v4   (:5173)
├── logistica-api/           # Django 6 + DRF 3.17 + SQLite    (:8000)
├── logistica-api_old/       # Deprecated — Django 6 (no dev)
├── clase-01..04/            # Course notes — DO NOT MODIFY
└── clase-sdd/               # SDD course notes — DO NOT MODIFY
```

No workspace tooling (npm workspaces, etc.). Each project is standalone.

---

## task-manager-backend

| | |
|---|---|
| Runtime | Node.js ESM (`"type": "module"`) |
| DB | PostgreSQL via Prisma 7 + `@prisma/adapter-pg` |
| Auth | bcrypt + jsonwebtoken (Bearer tokens sent from frontend) |
| Docs | Swagger at `http://localhost:3000/api-docs` |

**Commands:** AI may run `npx prisma migrate dev --name <name>`, `npx prisma studio`. Dev server is manual only.

**Never edit** `src/generated/prisma/` — auto-generated by Prisma. Edit `prisma/schema.prisma` instead.

**Keys:** `prisma.config.ts` (not prisma/schema.prisma alone), generated client output configured there.

---

## task-manager-frontend

| | |
|---|---|
| Framework | React 19 + TypeScript 6 |
| Bundler | Vite 8 |
| Styling | Tailwind v4 (via `@tailwindcss/vite` plugin — no config file) |
| Lint | `npm run lint` (ESLint 10) |
| Build | `npm run build` (runs `tsc -b` first) |

**Commits:** frontend stores JWT in `localStorage("token")` and sends via axios interceptor (`Authorization: Bearer`). 401 response clears token and redirects to `/login`.

**Dev server is manual only.**

---

## logistica-api (active Django project)

| | |
|---|---|
| Runtime | Python 3 + Django 6.0.5 |
| API | DRF 3.17.1 + SimpleJWT (JWT required by default on all endpoints) |
| DB | SQLite in dev (`config/settings/development.py`), PostgreSQL ready via psycopg2 |
| Docs | OpenAPI at `http://localhost:8000/api/v1/docs/` |
| Config | Settings split: `base.py` → `development.py` / `production.py` (selected via `DJANGO_SETTINGS_MODULE`) |

**Always activate venv first:** `source .venv/bin/activate` (Unix) or `.venv\Scripts\Activate.ps1` (Windows).

**`python manage.py runserver` is manual only.** All other commands (migrate, test, makemigrations) are AI-safe.

**Architecture:** 8 Django apps under `apps/` (authentication, warehouses, suppliers, customers, transport, products, routes, drivers, shipments). DRF ModelViewSets + drf-spectacular for OpenAPI schema.

**Methodology: SDD (Spec Driven Development):**
- 4 agents: Orchestrator → Spec → Implement → Validator
- Agents live in `.claude/agents/` — agent definitions
- Specs in `spec/`, docs in `docs/` (read `docs/architecture.md` + `docs/database-schema.md` before coding)

## Commands summary

```bash
# task-manager-backend
cd task-manager-backend
npx prisma migrate dev --name <name>   # AI allowed
npx prisma studio                      # AI allowed
npm run dev                            # MANUAL ONLY

# task-manager-frontend
cd task-manager-frontend
npm run lint                           # AI allowed
npm run build                          # AI allowed
npm run dev                            # MANUAL ONLY

# logistica-api
cd logistica-api
source .venv/bin/activate && python manage.py makemigrations   # AI allowed
source .venv/bin/activate && python manage.py migrate           # AI allowed
source .venv/bin/activate && python manage.py test              # AI allowed
source .venv/bin/activate && python manage.py test <app>        # single app
source .venv/bin/activate && python manage.py runserver         # MANUAL ONLY

# logistica-frontend — E2E
cd logistica-frontend
npm run e2e                            # Run all E2E tests
npm run e2e -- --grep "Warehouses"     # Run specific group
.\scripts\run-e2e.ps1                  # Auto-start servers + run (Windows)
./scripts/run-e2e.sh                   # Auto-start servers + run (Unix)
```
